Respuestas Teóricas - TodoList (Vue.js)

Ejercicio 2: ¿Qué es tasks.length? ¿Dónde se define en el template y en el script?

  • ¿Qué es? Es una propiedad nativa de JavaScript que devuelve el número de elementos que hay dentro de un array. En este caso, nos dice exactamente cuántas tareas se han añadido a la lista.
  • En el Template (HTML): Se utiliza dentro de la etiqueta <h1> usando la interpolación de Vue (dobles llaves): {{ tasks.length }} para mostrar el número de forma dinámica en la pantalla.
  • En el Script (JS): Viene del array tasks que está definido y se inicializa vacío dentro de la función data().

Ejercicio 3: ¿Qué hace esta línea de código?

data: () => ({ newTask: "", tasks: [] }),

Es una forma abreviada (usando una función flecha de JavaScript) de definir el estado reactivo o las variables iniciales de nuestro componente. Hace dos cosas:

  • Crea la variable newTask como una cadena de texto vacía. Aquí es donde se guardará automáticamente lo que el usuario escriba en el input.
  • Crea la variable tasks como un array vacío []. Aquí es donde se irán almacenando en forma de lista todas las tareas que el usuario vaya creando.

Ejercicio 4: Análisis del elemento <li>

<li class="task" v-for="(task, i) in tasks" :key="'task' + i" :class="{completed: task.completed}" @click="completeTask(task.text)"> {{task.text}} </li>
  • ¿Qué hace v-for?
    Es una directiva de bucle. Le indica a Vue: "Por cada tarea (task) que exista en la lista general (tasks), renderiza o dibuja un elemento <li> nuevo". Genera la lista visual.
  • ¿Qué es @click y por qué llama a completeTask(task.text)?
    @click es un "escuchador" de eventos. Detecta cuándo el usuario hace clic izquierdo con el ratón sobre ese elemento de la lista. Al hacer clic, ejecuta la función completeTask y le envía como parámetro el texto exacto de esa tarea (task.text) para que el código sepa específicamente cuál de todas las tareas debe marcar o desmarcar como completada.

Ejercicio 5: Identifica los dos métodos

Los métodos están definidos en el bloque methods: { ... } del script y son los siguientes:

  • createTask(): Se activa al enviar el formulario. Toma el texto que hay en newTask, construye un objeto nuevo indicando que la tarea no está completada (completed: false), lo añade al final del array tasks con .push() y, finalmente, vacía el input para que puedas escribir otra tarea.
  • completeTask(taskText): Se activa al hacer clic en una tarea. Recorre todo el array de tareas con un bucle for buscando aquella cuyo texto coincida con el que le han pasado. Cuando la encuentra, invierte su estado booleano (si completed era false, lo pasa a true, y viceversa) para tacharla o destacharla visualmente.
